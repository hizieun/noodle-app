# 5. 향후 개선 사항 및 업데이트 로드맵

이 문서는 프로젝트의 지속적인 발전을 위한 기술적 개선 사항과 추가 기능 아이디어를 정리합니다.

---

## 🛠 백엔드 및 인프라 개선

### 1. 크롤링 자동화 (배치 처리)
- **Crontab 활용**: 서버 환경에서 특정 시간(예: 매주 월요일 새벽)에 크롤러가 자동으로 실행되도록 설정합니다.
- **데이터 최신성 유지**: 정기적인 크롤링을 통해 폐업한 식당을 골라내고 새로운 맛집 데이터를 자동으로 갱신합니다.

### 2. 데이터베이스 및 API 서버 구축
- **DB 전환**: 현재의 `data.json` 방식에서 PostgreSQL 또는 MongoDB와 같은 데이터베이스 시스템으로 전환합니다.
- **REST API / GraphQL**: 프론트엔드에서 고정된 JSON 파일을 읽는 대신, 백엔드 서버에 API를 요청하여 실시간 데이터를 받아오도록 구성합니다.
- **성능 최적화**: 수천 개의 데이터를 효율적으로 처리하기 위한 인덱싱 및 캐싱 전략을 도입합니다.

---

## 🎨 프론트엔드 및 사용자 경험(UX) 개선

### 1. 브랜드 아이덴티티 강화
- **페이지 타이틀 변경**: 'Noodle' 대신 노포의 정체성을 더 잘 드러낼 수 있는 네이밍을 검토합니다.
  - *아이디어*: 세월의 맛, 노포지도, 백년식당, 찐노포 등
- **테마 및 배경**: 노포 특유의 따뜻하고 오래된 느낌을 줄 수 있는 배경 텍스트 및 전용 폰트, 빈티지한 다크 모드 테마를 적용합니다.

### 2. 지도 인터페이스 강화
- **지도 뷰 모드**: 현재의 그리드 리스트 외에, 지도 위에서 맛집 위치를 한눈에 볼 수 있는 인터랙티브 지도를 추가합니다.
- **내 주변 맛집**: 사용자의 위치 정보(Geolocation API)를 기반으로 현재 위치에서 가장 가까운 노포를 추천합니다.

---

## ✨ 신규 기능 아이디어

### 1. RAG 기반 맛집 추천 챗봇
- **AI 비서**: LLM(대규모 언어 모델)과 RAG(검색 증강 생성) 기술을 결합하여 사용자의 질문에 맞는 맛집을 추천합니다.
- **예시 질문**:
  - "성시경이 추천한 강남구 낙지볶음 맛집이 어디야?"
  - "비 오는 날 어울리는 종로구 노포 술집 알려줘."
  - "부모님 모시고 가기 좋은 깔끔한 노포 식당 추천해줘."

### 2. 사용자 리뷰 및 커뮤니티
- **직접 리뷰**: 사용자가 직접 사진과 리뷰를 올릴 수 있는 기능을 추가합니다.
- **즐겨찾기**: 마음에 드는 식당을 나만의 '노포 리스트'에 저장하는 기능을 제공합니다.
